Commandes Git indispensables

Gardez cette fiche à portée pour retrouver les actions essentielles, de l'initialisation au déploiement.

Configuration et découverte

Commencez par paramétrer Git pour votre machine puis inspectez les bases de votre dépôt.

Initialiser

git init

Crée un dépôt Git dans le dossier courant.

Cloner un projet

git clone <url>

Copie un dépôt distant tout en configurant l'origine.

Configurer son identité

git config --global user.name "Nom"

Définit le nom utilisé dans les commits.

Inspecter l'état

git status

Affiche les fichiers suivis, modifiés ou ignorés.

Construire des commits utiles

Structurez vos changements en petites unités cohérentes pour collaborer plus facilement.

Préparer

git add <fichier>

Ajoute un fichier suivi dans la zone de préparation.

Valider

git commit -m "message"

Enregistre les modifications préparées avec un message clair.

Modifier le dernier commit

git commit --amend

Complète ou corrige le commit précédent sans en créer un nouveau.

Mettre de côté

git stash push -m "expérimentation"

Sauvegarde des changements temporaires pour revenir sur une base propre.

Maîtriser les branches

Les branches permettent de travailler en parallèle et d'explorer sans casser la version principale.

Créer

git branch feature/auth

Prépare une nouvelle branche basée sur la branche courante.

Bascule rapide

git checkout feature/auth

Change la branche active dans l'arborescence de travail.

Fusionner

git merge feature/auth

Intègre la branche choisie dans la branche courante.

Réorganiser

git rebase main

Rejoue vos commits par-dessus une branche à jour pour une histoire linéaire.

Collaborer avec un dépôt distant

Synchronisez votre travail avec votre équipe et récupérez les dernières mises à jour.

Créer un lien distant

git remote add origin <url>

Associe un dépôt distant au dépôt local.

Récupérer les nouveautés

git fetch origin

Télécharge les branches distantes sans toucher à votre travail local.

Synchroniser proprement

git pull --rebase

Réapplique vos commits au-dessus des derniers changements pour éviter les merges inutiles.

Publier sa branche

git push -u origin feature/ui

Envoie vos commits et définit la branche distante par défaut.

Diagnostiquer

  • git log --oneline --graph
  • git diff main..feature/ui
  • git blame index.html

Bonnes pratiques

  • Commencez vos messages par un verbe à l'infinitif.
  • Synchronisez-vous avant de fusionner une branche.
  • Nettoyez vos branches une fois fusionnées.

Aller plus loin

  • git cherry-pick pour récupérer un commit.
  • git bisect pour retrouver l'origine d'un bug.
  • git reflog pour remonter dans l'historique local.
